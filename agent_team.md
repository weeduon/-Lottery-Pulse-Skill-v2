# 玄彩引擎 Agent 团队设计

## 1. 数据采集员 Data Harvester
职责：拉取快乐8 / 大乐透近 1000 期数据，保存原始文件，记录来源与时间。

## 2. 数据审计员 Data Auditor
职责：校验期号、日期、号码范围、重复号、缺失期。多源冲突时输出冲突清单。

## 3. 规则裁判 Rule Referee
职责：维护快乐8与大乐透规则、奖金条件、概率公式，负责中奖等级判断。

## 4. 统计分析师 Stat Analyst
职责：计算冷热、遗漏、和值、奇偶、大小、分区、连号、重号、尾数、AC 值。

## 5. 策略工程师 Strategy Engineer
职责：生成模拟组合策略，包括随机基准、冷热均衡、遗漏回归、分区覆盖、低相关组合。

## 6. 回测官 Backtest Judge
职责：用历史窗口回测策略，输出命中等级、投入、返奖、最大回撤、连亏期数。

## 7. 风险监察员 Risk Sentinel
职责：检查回答中是否出现“稳赚”“必中”“内部号”等误导话术，并强制改成研究/娱乐表达。

## 推荐运行顺序
Data Harvester → Data Auditor → Rule Referee → Stat Analyst → Strategy Engineer → Backtest Judge → Risk Sentinel

## 新增角色：每日选号官 Daily Picker

职责：

1. 每天基于最新历史数据生成若干组快乐8 / 大乐透模拟组合。
2. 为每组号码输出结构化理由：分区、奇偶、和值、高频、遗漏、重复控制。
3. 调用概率公式输出理论概率。
4. 强制声明：组合是娱乐研究，不是预测，不承诺收益。
5. 把每日组合写入 `outputs/`，方便第二天回测。

推荐流程：

```text
数据采集员 -> 数据审计员 -> 统计分析师 -> 每日选号官 -> 回测官 -> 风险监察员
```
