# AI预测模式设计参考

> 参考方向：彩虹多多这类彩票 APP 常见的“AI预测/智能选号”产品形态。由于未取得可核验的彩虹多多公开技术说明，本设计不声称复刻其真实算法，只借鉴其可能的产品表达方式。

## 产品定位

本项目中的“AI预测”应命名为：

- AI辅助选号
- AI组合评分
- AI模拟预测
- AI趋势参考

禁止命名为：

- 必中预测
- 稳赚模型
- 内部号
- 开奖算法破解

彩票开奖结果具有随机性，AI模块只能把历史统计、结构约束和组合评分包装成更容易理解的报告，不能制造确定性。

## 功能参考

| 模块 | 说明 | 本项目实现方式 |
|---|---|---|
| AI评分 | 给每个号码或组合打分 | 频率、遗漏、近期趋势、分区、奇偶、和值综合评分 |
| 热冷分析 | 展示高频、低频、当前遗漏 | 从历史 CSV 计算出现次数和遗漏期数 |
| 趋势预测 | 观察最近 N 期变化 | 最近 20/50/100 期加权，不直接宣称预测 |
| 智能选号 | 输出多组号码 | 按评分加权抽样，再进行结构过滤 |
| 置信度 | 给组合一个参考等级 | 输出“结构置信度”，不等于中奖概率 |
| 命中概率 | 展示理论概率 | 使用组合数学计算，不因 AI评分改变 |
| 回测复盘 | 检验策略历史表现 | 对每期模拟候选组合进行命中分布统计 |

## AI评分维度

### 1. 高频分

号码在历史样本中出现越多，基础频率分越高。

```text
hot_score = freq(number) / max_freq
```

### 2. 遗漏分

号码距离上次出现越久，遗漏分越高。

```text
miss_score = omission(number) / max_omission
```

### 3. 近期趋势分

近 N 期出现情况加权，越近权重越高。

```text
trend_score = weighted_recent_freq(number)
```

### 4. 结构平衡分

组合层面判断：

- 大乐透前区三区分布是否合理；
- 大乐透后区是否避免极端结构；
- 快乐8四区分布是否过度集中；
- 奇偶比例是否极端；
- 和值是否落在历史常见区间。

### 5. 去重与相关性

每日多组号码之间应降低重复率，避免生成 5 组看起来像复制粘贴的号码。虽然随机数并不在乎，但用户至少能看出系统没睡着。

## 评分公式建议

```text
number_score = 0.35 * hot_score
             + 0.30 * miss_score
             + 0.25 * trend_score
             + 0.10 * random_noise
```

组合评分：

```text
combo_score = avg(number_score)
            + structure_bonus
            - extreme_penalty
            - duplicate_penalty
```

## 输出格式

AI预测报告应包含：

| 字段 | 说明 |
|---|---|
| 组别 | 第几组 |
| 号码 | 大乐透分前区/后区；快乐8显示所选号码 |
| AI评分 | 0-100 分 |
| 结构置信度 | 低 / 中 / 高，只表示结构均衡程度 |
| 选择理由 | 热冷、遗漏、趋势、分区、和值等 |
| 理论概率 | 真实组合概率，必须显示 |
| 风险提示 | 不构成投注建议 |

## 推荐命令

```bash
python scripts/ai_predict.py --game dlt --input data/dlt_1000.csv --groups 5 --out outputs/dlt_ai_predict.md
python scripts/ai_predict.py --game kl8 --input data/kl8_1000.csv --choose-k 10 --groups 5 --out outputs/kl8_ai_predict.md
```

## 风控文案

推荐固定写入报告底部：

```text
AI评分只表示组合结构与历史统计特征，不表示中奖概率提升。彩票开奖结果具有随机性，请控制预算，勿追投。
```
